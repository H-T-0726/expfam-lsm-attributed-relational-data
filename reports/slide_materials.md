## スライド7：提案手法の全体像

**スライドタイトル**：提案手法：指数型分布族への統一的拡張

**メインメッセージ**：  
> 変えるのは観測分布、残すのは潜在構造

**スライド上の文章**：

- 従来手法（Mikawa 2024）の骨格をそのまま維持
  - 潜在変数の事前分布：z_i ~ N(0, I_k)
  - 自然パラメータの形式：η^X = f_l^T z_i、η^Y = w_0 + w z_i^T z_j
  - 推定方法：MCEM + Laplace近似
- 変更点：観測分布 ExpFam_X、ExpFam_Y を分析者が指定
  - Gaussian、Bernoulli、Poisson をデータ型に合わせて選択可能

**口頭説明**：  
「先行研究のモデルと推定方法はそのままに、観測データの分布族だけを柔軟に変えられるようにしました。この変更により、属性データが二値でも、関係データがカウントデータでも、一つの枠組みで対応できます」

## スライド8：提案モデルの定式化

**スライドタイトル**：生成モデルの定式化

**メインメッセージ**：属性と関係のそれぞれに指数型分布族を導入

**載せる式**：

```
(4)   y_ij | z_i, z_j ~ ExpFam_Y(η_ij^Y),   η_ij^Y = w_0^Y + w^Y z_i^T z_j   (i < j)

(5)   x_il | z_i ~ ExpFam_X(η_il^X),         η_il^X = f_l^T z_i
```

**パラメータ**：θ = {F ∈ R^{d×k},  w_0^Y,  w^Y ∈ R}（Gaussian X のとき対角 Σ も推定）

**スライド上の箇条書き**：
- ExpFam_Y に Bernoulli を指定 → 式(4) は従来手法の式(1) と一致
- ExpFam_X に Gaussian を指定 → 式(5) は従来手法の式(2) と一致
- ExpFam_X と ExpFam_Y は独立に指定可能

**口頭説明**：  
「式の形自体は先行研究と全く同じです。ε_ij = σ(w_0 + w z_i^T z_j) というリンク関数が、指数型分布族の自然パラメータに対応します。Bernoulli を選べばシグモイドが、Poisson を選べば指数関数が、確率モデルとして使われます」

## スライド9：推定方法

**スライドタイトル**：MCEM + Laplace近似による推定

**メインメッセージ**：推定の枠組みは先行研究と同一、精度行列が一般化

**スライド上の文章**：

E-step（各 z_i の事後分布を更新）：
- Laplace近似：q_i(z_i) = N(m_i, A_i^{-1})
- 精度行列（原稿 式(6)）：
  - A_i = I_k + F^T V_X(m_i) F + (w^Y)^2 Σ_{j≠i} A_Y''(η_ij^Y) z_j z_j^T
  - ← 第2項が提案手法の一般化の核心（ExpFam_X に応じて変化）
- L 個の MC サンプルを生成

M-step（θ を Q 関数最大化で更新）：
- F：Gaussian X → 閉形式解、非Gaussian X → Adam 勾配上昇
- w_0^Y, w^Y：常に Adam
- k 選択：BIC

**口頭説明**：  
「先行研究では精度行列の属性側項が F^T Σ^{-1} F（Gaussian 固有）でした。提案手法では、これを分布族の分散関数 A_X'' を使った V_X(m_i) に置き換えることで、任意の指数型分布族に対応します」

## スライド10：実験設定

**スライドタイトル**：人工データを用いた評価実験

**メインメッセージ**：3種類の分布族組み合わせで有効性を検証

**スライド上の文章**：

| シナリオ | ExpFam_X（属性） | ExpFam_Y（関係） |
|---------|--------------|--------------|
| A | Poisson | Bernoulli |
| B | Gaussian | Poisson |
| C | Bernoulli | Gaussian |

- n=150（基本値）、d=15、k*=3（真の潜在次元）、10試行
- 評価指標：RMSE(Z)（Procrustes回転適用後）
- 追加：Control条件（X=Gauss, Y=Bern）で先行研究[1]と比較

**口頭説明**：  
「先行研究が対応していない分布族の組み合わせを3種類設定しました。先行研究との公平な比較のため、先行研究が直接適用できる条件でも評価しています」

## スライド13：実験結果(3) — 分布族ミスマッチ

**スライドタイトル**：誤った分布族の指定が推定精度を大幅に悪化させる

**使う図**：figures/fig1b_misspecification_color.png

**図のキャプション案**：  
「図1(b)：分布族の誤指定が RMSE(Z) に与える影響（正しい指定 = 1.0 とした悪化倍率、破線：基準線）」

**図の見方（口頭）**：  
「水色のバー（斜め線）が X 側のみ誤指定した場合の最大値、橙のバー（クロス）が Y 側のみ誤指定した場合の最大値、灰色のバー（横線）が従来手法相当（X=Gaussian, Y=Bernoulli 固定）の悪化倍率を示します」

**口頭説明**：  
「誤った分布族を指定すると、どのシナリオでも大幅な精度劣化が生じます。たとえばシナリオ A では、X の分布族を誤指定すると最大 3.41 倍も悪化します。X を全く使わない場合（1.25倍）よりも、誤った分布族で使う方が悪いという興味深い結果も得られました」

**注意事項**：  
「図中の最大値は 23.6×（灰色バー、Scen.C）です。原稿本文に記載のある 41.5× は図中にバーがない条件（X・Y 両方誤指定の最大値）に対応するため、質疑で聞かれた場合は説明が必要」

**結果の一言まとめ**：  
「誤った分布族指定は最大 23.6 倍以上の精度劣化を引き起こす」

## スライド14：考察

**スライドタイトル**：考察

**メインメッセージ**：正確な分布族指定が推定性能の鍵

**スライド上の文章**：

有効性の確認：
- BIC が3種類の分布族組み合わせすべてで正しい潜在次元を選択
- n 増加に伴う一貫した精度改善
- 先行研究と同条件で同等性を確認

注目すべき発見：
- シナリオ A: X の誤指定（最大3.41×）が Y の誤指定より影響大
- シナリオ C: Y=Gaussian が支配的なため、X の分布指定の影響が小さい
  - No X (X を使わない) と提案手法が同等 → Y 信号が強い場合の特性

限界：
- 実装に数値的課題が残っている（精度行列の定数係数、影響未定量化）
- 実データへの適用・Categorical 分布は今後の課題

## スライド15：まとめ

**スライドタイトル**：まとめ

**メインメッセージ**：指数型分布族への拡張で多様なデータ型に対応可能なモデルを実現

**スライド上の文章**：

提案：
- 関係データ・属性データの双方を指数型分布族で記述する潜在構造モデル
- 潜在構造の枠組みを維持しつつ、観測分布を柔軟に指定可能

実験で確認：
1. BIC による潜在次元の自動選択（3シナリオで成功）
2. n 増加に伴う推定精度の改善（49%〜62%削減）
3. 正確な分布族指定の重要性（誤指定で最大 23.6 倍以上の悪化）
4. 先行研究との同等性（同条件でRMSE(Z)差 < 0.001）

今後の課題：
- 実データへの適用
- Categorical 分布の実装
- パラメータ推定アルゴリズムの精緻化
